## 功能特性

### 🚀 核心功能

- **实时日志流** - 自动捕获并展示游戏内的 stdout/stderr 输出，支持日志分级显示
- **远程代码执行** - 在客户端/服务端动态执行 Python 代码，无需重启游戏
- **多设备管理** - 同时连接多个游戏客户端，独立管理每个会话的日志
- **跨平台支持** - 支持 PC 端和移动端设备的跨设备调试

### 💡 开发者体验

- **零配置连接** - 基于 UDP 广播的自动服务发现，无需手动配置 IP
- **现代化界面** - 基于 WebView2 的原生桌面应用，支持深色/浅色主题
- **代码高亮** - Python 代码编辑器内置语法高亮
- **可调整布局** - 支持拖拽调整面板大小，适应不同工作流

### 🔧 技术亮点

- **非阻塞网络** - 异步 TCP/UDP 通信，不影响游戏性能
- **日志缓存** - 智能日志缓存机制，防止内存溢出
- **会话隔离** - 每个客户端独立的日志会话，互不干扰
- **断线重连** - 自动重连机制，网络波动不影响调试

## 快速开始

### 环境要求

- **PC 端**: Windows 10/11 (x64)，需安装 WebView2 Runtime（通常已预装）
- **移动端**: 网易我的世界基岩版（支持 Mod 加载）
- **网络**: PC 和移动设备需处于同一局域网

### 使用步骤

#### 1. 启动 PC 控制台

在 Windows 电脑上双击运行 `MCDevConsole.exe`，控制台将自动启动并监听 UDP 端口 18232。

#### 2. 配置并上传 Mod

1. 打开 `mods/MCDevConsole` 目录
2. 修改 `manifest.json` 中的 UUID（避免与其他 Mod 冲突）
   ```json
   {
     "uuid": "your-unique-uuid-here",
     ...
   }
   ```
3. 将整个 `MCDevConsole` 文件夹上传至网易开发者平台
4. 提交机审或进入自测环境

#### 3. 移动端加载 Mod

1. 在移动设备上启动网易我的世界
2. 进入游戏设置，加载已上传的 MCDevConsole Mod
3. 创建或进入世界，Mod 将自动激活

#### 4. 自动连接

- 确保 PC 和移动设备连接到**同一个 Wi-Fi 网络**
- Mod 会通过 UDP 广播自动发现 PC 控制台
- 连接成功后，左侧设备列表将显示移动设备信息
- 日志将实时同步到控制台

> **提示**: 如果无法自动连接，请检查防火墙设置，确保允许 UDP 18232 端口通信。

```

│                     MCDevConsole Desktop                     │

│  │  (HTML/JS/CSS) │  │  (Protocol)  │  │  (TCP/UDP)      │ │
│  └────────────────┘  └──────────────┘  └─────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              ↕ TCP (18232+)
                              ↕ UDP Broadcast (18232)
┌─────────────────────────────────────────────────────────────┐
│                  Minecraft Bedrock Edition                   │
│  ┌────────────────────────────────────────────────────────┐ │
│  │              MCDevConsole Mod (Python 2)               │ │
│  │  ┌──────────────┐  ┌──────────────┐  ┌─────────────┐ │ │
│  │  │ Log Proxy    │  │ Network      │  │ Code Exec   │ │ │
│  │  │ (stdout/err) │  │ (TCP Client) │  │ (eval/exec) │ │ │
│  │  └──────────────┘  └──────────────┘  └─────────────┘ │ │
│  └────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 通信协议

**数据包格式** (Big-Endian)
```
┌──────────┬──────────┬─────────────────┐
│ Type ID  │  Size    │      Data       │
│ (2 bytes)│ (4 bytes)│   (N bytes)     │
└──────────┴──────────┴─────────────────┘
```

**消息类型**
- `TYPE_STDOUT_LOG (1)` - 标准输出日志
- `TYPE_STDERR_LOG (2)` - 错误输出日志
- `TYPE_EXEC_CLIENT (3)` - 客户端代码执行
- `TYPE_EXEC_SERVER (4)` - 服务端代码执行

## 使用指南

### 基础操作

#### 1. 设备连接

启动控制台后，Mod 会自动通过 UDP 广播发现服务。左侧面板显示所有已连接的设备：

- **在线状态** - 绿色圆点表示在线，灰色表示离线
- **设备信息** - 显示设备名称、端点地址和平台信息
- **最后活动** - 显示最后一次日志时间

点击设备卡片可切换当前会话。

#### 2. 日志查看

中央面板实时显示选中设备的日志流：

- **日志分级** - 不同级别的日志使用不同颜色标记
  - `INFO` - 白色（默认）
  - `WARN` - 黄色
  - `ERROR/STDERR` - 红色
  - `DEBUG` - 青色
  - `SUCCESS` - 绿色

- **日志缓存** - 最多保留 2000 条日志，超出时自动清理
- **自动滚动** - 新日志到达时自动滚动到底部

#### 3. 代码执行

下方面板用于执行 Python 代码：

1. **选择目标** - 点击 "客户端" 或 "服务端" 按钮选择执行环境
2. **编写代码** - 在代码编辑器中输入 Python 代码
3. **执行** - 点击按钮执行

**示例代码**
```python
# 获取游戏版本
print("Game Version: 1.20.0")

# 执行游戏命令
import mod.server.extraServerApi as serverApi
api = serverApi.GetEngineCompFactory()
print("Server API loaded")

# 列表操作
items = [1, 2, 3, 4, 5]
print("Sum:", sum(items))
```

### 核心模块说明

#### NetworkServer (C++)

负责 TCP 服务器实现，接收来自 Mod 的日志和命令。

**关键类**
- `NetworkServer` - TCP 服务器主类
- `Protocol` - 数据包序列化/反序列化

#### NETSystem (Python)

Mod 端的网络通信模块，实现 UDP 自动发现和 TCP 连接。

**关键方法**
- `start()` - 启动网络系统
- `send_stdout(message)` - 发送标准输出日志
- `send_stderr(message)` - 发送错误日志
- `close()` - 关闭连接

#### LoggingProxySystem (Python)

拦截并代理 Python 的 stdout/stderr，实现日志捕获。

**关键方法**
- `startProxy()` - 启动日志代理
- `updateCachedLogs()` - 发送缓存的日志

### 性能优化

#### 已实现的优化

1. **非阻塞网络** - 所有网络操作在独立线程中执行，不阻塞游戏主线程
2. **日志缓存** - 连接建立前的日志被缓存，连接后批量发送
3. **智能清理** - 日志超过 2000 条时自动清理最早的 500 条
4. **锁优化** - 最小化锁持有时间，避免死锁

#### 性能指标

- **日志延迟** - < 50ms
- **内存占用** - 基础 ~50MB，日志缓存 ~10MB/1000条
- **网络带宽** - 平均 < 100KB/s
- **CPU 占用** - < 1% (空闲状态)

## 故障排除

### 常见问题

**Q: 控制台无法发现游戏设备**

A: 检查以下几点：
- 确保游戏和控制台在同一网络
- 检查防火墙是否阻止 UDP 18232 端口
- 尝试重启游戏和控制台

**Q: 日志显示延迟很高**

A: 可能原因：
- 网络连接不稳定，检查网络延迟
- 日志输出过于频繁，考虑减少日志量
- 检查 CPU 占用是否过高

**Q: 代码执行失败**

A: 检查以下几点：
- 确保代码语法正确（Python 2.7 语法）
- 检查导入的模块是否存在
- 查看错误日志了解具体错误信息

### 调试技巧

1. **启用详细日志** - 在 Mod 配置中启用 DEBUG 级别日志
2. **检查网络连接** - 使用 `netstat` 检查 TCP 连接状态
3. **查看系统日志** - 检查游戏的 `debug.log` 文件
